Prescriptive Traceability v0
Objetivo

La trazabilidad prescriptiva describe el flujo de información desde la captura de datos hasta la generación de acciones y reportes. Su propósito es identificar un problema, analizar sus causas y recomendar acciones para su solución.

Flujo de la trazabilidad
1. Imagen / Sensor / Satélite simulado / Clima / Historial

El proceso inicia con la recolección de información proveniente de distintas fuentes:

Imágenes capturadas por cámaras o drones.
Sensores instalados en el área de monitoreo.
Información proveniente de un satélite simulado.
Datos climáticos (temperatura, humedad, lluvia, viento, etc.).
Historial de eventos registrados anteriormente.

Estas fuentes constituyen la entrada principal del sistema.

↓

2. Evidencia

Los datos recopilados son procesados para generar evidencias que respalden la existencia de un problema.

Ejemplos:

Imagen con presencia de daño.
Lectura anormal de un sensor.
Registro de lluvia intensa.
Eventos similares ocurridos anteriormente.

↓

3. Riesgo

Con base en las evidencias se calcula el nivel de riesgo asociado.

Ejemplos:

Bajo
Medio
Alto
Crítico

↓

4. Alerta

Si el riesgo supera un umbral establecido, el sistema genera una alerta indicando el problema detectado y su ubicación.

↓

5. Recomendación

El sistema propone acciones para reducir o eliminar el riesgo detectado.

Ejemplos:

Realizar inspección en campo.
Aplicar mantenimiento preventivo.
Reemplazar un componente.
Programar una revisión técnica.

↓

6. Acción

Se registra la acción que realmente fue ejecutada por el responsable.

Ejemplos:

Inspección realizada.
Reparación completada.
Monitoreo continuo.
Acción pendiente.

↓

7. Historial

Todas las acciones quedan almacenadas para mantener la trazabilidad del evento y facilitar futuras consultas.

El historial permite conocer:

Fecha del evento.
Evidencias registradas.
Riesgo detectado.
Acciones recomendadas.
Acciones ejecutadas.
Estado actual.

↓

8. Reporte

Finalmente, el sistema genera un reporte prescriptivo que resume todo el proceso.

El reporte responde las siguientes preguntas:

¿Qué problema se detectó?
¿Dónde ocurrió?
¿Qué evidencia lo respalda?
¿Cuál fue el nivel de riesgo?
¿Qué recomendación se emitió?
¿Qué acción se ejecutó?
¿Qué quedó pendiente?
